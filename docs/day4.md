# Day 4 - Python 기반 서비스 상태 감지 구현

Day4 에서는 운영 자동화의 첫 단계인 "상태 감지"를 구현했다.

EC2에서 실행 중인 서비스의 `health` 엔드포인트를
Python 스크립트가 주기적으로 호출하고,
HTTP 상태 코드를 기준으로 정상/비정상 여부를 코드가 판단하도록 구성했다.

요청 실패, 타임아웃, 네트워크 오류 등은 모두 "Down"상태로 처리하고,
결과는 로그 파일로 기록해 이후 알림이나 자동 조치로 확장할 수 있는 구조를 만들었다.

이 단계까지 완료하면서,
운영 대상 서비스가 언제부터 정상/비정상 상태였는지를
사람이 아니라 코드가 판단할 수 있는 기반이 마련됐다.
